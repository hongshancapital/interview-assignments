## 设计思路

根据要求，设计思路如下：

1.长转短，有长度限制，需要有一种算法尽量避免冲突，经过查询资料，总结几种常见算法：ID自增+62进制编码、摘要算法。

Demo使用摘要算法

2.短转长，由于摘要算法不可逆性，所以需要把数据持久化，采用KV结构存储

3.关于存储：微服务架构可以考虑Redis+MySQL组合存储，单节点架构可以选择LRUCache+MySQL。

Demo使用LRUCache
