## 一、设计思路

1.将接收的请求中的长域名映射为一个自增数字id；

2.将数字id压缩为8位的字符串；

3.压缩过程可以使用10进制转62进制的方式实现；

4.为了控制62进制长度为8位，那数字id长度不能超过15位；

5.使用线程安全的Hashmap存储62进制id和长域名的对应关系，避免并发情况下数据出现问题；
