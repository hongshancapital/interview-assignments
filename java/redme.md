设计思路： 
	1、生成短链接方法：
	思路一：
		1、可以将方法传入的长链接按照一定规则，比如雪花算法生成指定长度的含字母和数字混合的字符串，作为短链接返回，
		2、然后将短链接字符串和长链接字符串保存为一个对象的2个属性或者map形式(key是短链接，value是长链接)，保存在数据库表或者内存里
	思路二：
		1、生成短链接的方法只是指定长度含字母和数字混合的随机字符串
		2、保存方式同上
	这里才有了思路二

	2、获取短链接对应的长链接方法：
		1、用传入的短链接，去掉http://或https://前缀的部分作为id，查询保存的内存或数据库对象
		2、返回长链接字符串的结果
